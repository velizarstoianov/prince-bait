# 👑 Prince Bait

> **Waste scammers' time. Protect real people.**

Prince Bait is an open-source AI-powered **419 scam baiting agent** that automatically engages advance-fee fraudsters using believable, AI-generated personas. Instead of blocking scam emails, Prince Bait keeps scammers occupied with convincing conversations, reducing the time they have available to target real victims.

Originally developed as a university diploma thesis, Prince Bait has evolved into a modern, provider-agnostic LLM application supporting Anthropic Claude, OpenAI, Ollama, and multiple email providers through a clean web interface.

> ⚠️ **For anti-fraud and research purposes only.** Prince Bait never uses real identities, financial information, or payment details. Every persona and banking detail generated by the system is entirely fictional.

---

## Why?

Advance-fee ("419") scams rely on spending time with potential victims.

Prince Bait reverses that equation.

Instead of deleting scam emails, the agent engages scammers with believable but perpetually stalled conversations inspired by the classic **419eater** scam-baiting methodology.

The objective is simple:

- waste scammers' time
- reduce their productivity
- make large-scale fraud less profitable
- provide an open research platform for scam analysis

Prince Bait is **not** designed to trick innocent people or automate phishing. It exists solely to engage known scammers.

# Quick Start

## Requirements

- Python 3.11+
- Anthropic, OpenAI, or Ollama
- (Optional) Email account for automated baiting

## Installation

```bash
git clone https://github.com/yourusername/prince-bait.git

cd prince-bait

pip install -r requirements.txt
```

Copy the environment template:

```bash
cp .env.example .env
```

Add your preferred API key:

```text
ANTHROPIC_API_KEY=...
```

Then start the application:

```bash
python -m uvicorn app:app --reload --port 8000
```

Open:

```
http://127.0.0.1:8000
```

The SQLite database (`scambaiter.db`) is created automatically on first launch.

# Core Components

## Agent

The agent orchestrates conversations using:

- conversation history
- personas
- tool calls
- configurable LLM providers

Its behavior is guided by a system prompt based on the 419eater scam-baiting methodology.

---

## LLM Providers

Prince Bait supports multiple AI providers through a unified interface.

Current providers include:

- Anthropic Claude
- OpenAI
- Ollama

Adding new providers only requires implementing the provider interface.

---

## Tool System

The agent can invoke tools during conversations.

Current tools include:

- Detect banking intent
- Generate fake identities
- Spawn additional characters

The tool architecture is fully extensible.

---

## Identity Generation

Prince Bait can operate in three modes:

### Fake

Uses entirely fabricated identities.

Recommended for automated baiting.

### Real

Uses only the user's supplied name/email.

No financial information is fabricated.

### Ask

Prompts for identity mode when creating a new thread.

---

## Email Automation

Prince Bait supports both API-based and standard mail connections.

Supported methods include:

- IMAP / SMTP
- Gmail OAuth
- Microsoft OAuth

Email credentials are encrypted using Fernet before storage.

---

## Database

Supports:

- SQLite (default)
- PostgreSQL

Stored entities include:

- conversation threads
- personas
- participants
- messages
- mail accounts

# REST API

Main endpoints include:

| Endpoint | Description |
|-----------|-------------|
| `POST /chat` | Run the AI agent |
| `GET /config` | Configuration |
| `GET /threads` | List conversations |
| `GET /threads/{id}` | Retrieve thread |
| `POST /threads` | Create thread |
| `POST /process-emails` | Process unread emails |
| `GET /mail/accounts` | List mail accounts |
| `POST /mail/accounts` | Add account |
| `POST /mail/accounts/{id}/test` | Test account |
| `POST /mail/accounts/{id}/activate` | Activate account |
| `DELETE /mail/accounts/{id}` | Remove account |

# Security

Prince Bait is designed with several safety principles:

- No real banking information is generated
- Fake identities are completely fabricated
- Mail credentials are encrypted at rest
- Secrets are never returned through the API
- API keys can be provided via environment variables or the UI

Never commit:

- `.env`
- databases
- OAuth credentials
- tokens
- encryption keys
